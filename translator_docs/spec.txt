L-ASP TRANSLATOR SPECIFICATION

1. FUNCTIONALITY

The translator converts an L program into an ASP program.

Manual demos of such translations can be found at:
https://github.com/iensen/LtoASPtranslator/tree/master/translator_docs/demos
Automated examples are located at:
https://github.com/iensen/LtoASPtranslator/tree/master/src/translator_examples
In these two folders, an L file has extension .l, whereas an ASP file has extension .lp (unsorted) or .sp (sorted).

L spec is given at:
https://github.com/iensen/LtoASPtranslator/blob/master/Lspec/Lspec.pdf
Sorted ASP manual is provided at:
https://github.com/iensen/sparc/blob/master/User_Manual/Sparc_Manual.pdf

2. MODULES

The translator comprises three modules: parser, transformer, and unparser.

2.1. PARSER

The parser reads an L program and generates a list of L parse trees. The parser is introduced at:
https://github.com/iensen/LtoASPtranslator/blob/master/README.txt

This L parser is based on a generic parser whose spec is located at:
https://github.com/iensen/genparser/blob/master/docs/main/astgen.pdf
The lexicon in use is:
https://github.com/iensen/LtoASPtranslator/blob/master/src/lexicon
And the grammar is:
https://github.com/iensen/LtoASPtranslator/blob/master/src/grammar

2.2. TRANSFORMER

The transformer's input is the parser's output: a parsed L program. The transformer produces an equivalent list of ASP parse trees. The format of these trees is specified in:
https://github.com/iensen/genparser/blob/master/docs/main/astgen.pdf
The lexicon used is:
https://github.com/iensen/LtoASPtranslator/blob/master/src/lexicon_ASP
And the grammar is:
https://github.com/iensen/LtoASPtranslator/blob/master/src/grammar_ASP

2.3. UNPARSER

Given a parsed ASP program from the transformer, the unparser generates the corresponding ASP program.
